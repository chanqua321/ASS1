EduAI (Assigment1)
Luồng nghiệp vụ và kiến trúc code

Mục đích: Tài liệu giúp thành viên nhóm đọc hiểu cách hệ thống hoạt động — từ góc nhìn nghiệp vụ và map tới file code tương ứng.

Phiên bản: Theo codebase hiện tại — ASP.NET Core 8 MVC, SQL Server LocalDB, Chat RAG với Ollama (local).

1. Hệ thống là gì?

EduAI là ứng dụng web quản lý tài liệu học tập kèm trợ lý AI hỏi–đáp theo nội dung tài liệu đã upload (kỹ thuật RAG — Retrieval-Augmented Generation).

  • WF1 — Quản lý tài liệu: Giáo viên upload PDF/DOCX/PPT → trích text → chia chunk → embedding → lập chỉ mục (Indexed).
  • WF2 — Chat RAG: Người dùng hỏi trong phiên chat → hệ thống tìm đoạn tài liệu liên quan → AI (Ollama) trả lời, có thể kèm trích dẫn nguồn.
  • Quản trị: Admin tạo môn/chương, gán giáo viên, xem dashboard và audit log.
  • Mở rộng: Tóm tắt AI tài liệu, sinh quiz trắc nghiệm, export quiz PDF.

5. WF1 — Quản lý và lập chỉ mục tài liệu

Controller: Web/Controllers/DocumentsController.cs
Service: BusinessLogic/Logic/DocumentService.cs

5.1. Upload (Teacher)

  1. Validate định dạng (pdf, docx, pptx) và chọn Subject + Chapter.
  2. Lưu file vật lý vào App_Data/uploads/{guid}.ext.
  3. Insert bản ghi Document (Status = Pending).
  4. Gọi ProcessDocumentAsync để index.

5.2. Pipeline index (ProcessDocumentAsync)

  1. Đặt Status → Processing (UI hiển thị đang xử lý).
  2. Xóa chunk cũ (nếu reindex).
  3. DocumentTextExtractor — trích text theo loại file.
  4. ChunkingService — chia đoạn (~800 ký tự, overlap 100).
  5. MockEmbeddingService — tạo vector (demo; production có thể thay OpenAI/Azure).
  6. Lưu DocumentChunk + DocumentEmbedding vào DB.
  7. Đặt Status → Indexed hoặc Failed (kèm ErrorMessage).
  8. DocumentSummaryService — tóm tắt AI (bổ trợ; lỗi không làm fail quá trình index).
Trạng thái DocumentÝ nghĩa
PendingVừa upload, chờ xử lý
ProcessingĐang trích text / chunk / embed
IndexedSẵn sàng cho Chat RAG và Quiz
FailedLỗi — xem ErrorMessage trên UI

6. WF2 — Chat RAG (hỏi đáp theo tài liệu)

Controller: Web/Controllers/ChatController.cs
Service: BusinessLogic/Logic/ChatService.cs

6.1. Điều kiện sử dụng

  • Tài liệu môn học phải ở trạng thái Indexed.
  • Student đăng ký môn qua POST /Chat/Enroll (bảng SubjectEnrollments).
  • Tạo phiên chat và chọn SubjectId để giới hạn phạm vi tìm kiếm.
  • Bật Ollama local (ollama serve, model llama3.2) để có câu trả lời AI thật.

6.2. Luồng gửi tin nhắn (POST /Chat/Send — JSON)

  1. Lưu hoặc tạo ChatSession.
  2. Nếu câu xã giao (chào, cảm ơn…) → trả lời template, không gọi AI.
  3. Lưu tin nhắn User vào DB.
  4. Lấy lịch sử chat gần nhất (MaxHistoryMessages = 6).
  5. RetrievalService: embed câu hỏi → so cosine similarity với chunk → lấy top K (TopK = 5).
  6. RagAnswerGenerator: POST tới Ollama {BaseUrl}/chat/completions hoặc fallback trích đoạn local nếu AI lỗi/tắt.
  7. Lưu tin nhắn Assistant + MessageCitation (file, chunk, score).
  8. Trả JSON ChatSendResponse cho giao diện.

Câu hỏi tóm tắt dùng nhánh RetrieveForSummaryAsync (SummaryTopK = 12, lấy chunk đều theo tài liệu).

11. Hướng dẫn demo nhanh

  1. Chạy dotnet run trong thư mục Web (SQL LocalDB + migrate tự động).
  2. Chạy Ollama: ollama pull llama3.2 và ollama serve.
  3. Đăng nhập admin@gmail.com / 123 → tạo môn, teacher, student.
  4. Teacher upload tài liệu → đợi trạng thái Indexed.
  5. Student đăng ký môn → vào Chat → hỏi theo nội dung tài liệu.
